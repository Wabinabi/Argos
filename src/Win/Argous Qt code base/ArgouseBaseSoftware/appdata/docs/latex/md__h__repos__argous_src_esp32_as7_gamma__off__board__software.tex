The Off-\/\+Board software, otherwise known as the Base Station Software, refers to the project desktop software. It is a Windows 10 application that processes the data gathered by the drone platform for visualisation and export. Developed on Qt Creator, the application has to capacity to be further developed utilising this software. Environment and set-\/up instructions can be found on the \href{index.html}{\texttt{ main page.}}

\hypertarget{md__h__repos__argous_src_esp32_as7_gamma__off__board__software_autotoc_md10}{}\doxysection{Importing Drone Data}\label{md__h__repos__argous_src_esp32_as7_gamma__off__board__software_autotoc_md10}
The on-\/board software does not fully process all captured data to reduce on-\/board processing requirements. This is done by the off-\/board or base station software using the outputs from the drone.

Data recorded by the drone is saved to the Micro\+SD card as part of the electronics package. These files are {\ttfamily as7.\+log} and{\ttfamily data.\+csv}.

To import data from the drone\+:


\begin{DoxyEnumerate}
\item Plug in the Micro\+SD Card to this computer using an adaptor
\item Press “\+Import”
\item Locate the {\bfseries{folder}} that the files are located in. This is typically found under “\+My Computer” and is the SD card itself
\item Press “\+Select Folder” and the data will begin importing.
\item There should be a prompt saying “\+Import Successful” if the import is successful.
\end{DoxyEnumerate}



{\itshape The Base Station Software highlighting the import button}

Imports can also be done on recently imported files to re-\/generate them.


\begin{DoxyEnumerate}
\item Press on the recent file under “\+Recent Imports”
\item Press “\+Import”
\item There should be a prompt saying “\+Import Successful” if the import is successful.
\end{DoxyEnumerate}\hypertarget{md__h__repos__argous_src_esp32_as7_gamma__off__board__software_autotoc_md11}{}\doxysubsubsection{Opening Drone Details}\label{md__h__repos__argous_src_esp32_as7_gamma__off__board__software_autotoc_md11}
Drone details cannot be opened unless a file has been imported. Drone configuration is read from the import itself, and therefore a trip must be imported to view configuration details.\hypertarget{md__h__repos__argous_src_esp32_as7_gamma__off__board__software_autotoc_md12}{}\doxysubsubsection{Editing Drone Details}\label{md__h__repos__argous_src_esp32_as7_gamma__off__board__software_autotoc_md12}
The drone configuration can be edited through {\ttfamily as7.\+config} or through the base station software. This can be edited directly and saved or reset.



{\itshape The save button (scroll down) can be used to save the drone configuration and details}\hypertarget{md__h__repos__argous_src_esp32_as7_gamma__off__board__software_autotoc_md13}{}\doxysubsubsection{Opening Trip Details Without Importing}\label{md__h__repos__argous_src_esp32_as7_gamma__off__board__software_autotoc_md13}
The trip details can be opened and viewed if there has been no import. The previous import is loaded if no new import has been found.

\hypertarget{md__h__repos__argous_src_esp32_as7_gamma__off__board__software_autotoc_md14}{}\doxysection{Viewing Trip Details}\label{md__h__repos__argous_src_esp32_as7_gamma__off__board__software_autotoc_md14}
The details of the trip can be viewed by pressing on “\+View Trip Data” on the home page.



{\itshape View Trip Data can be seen under the import box}

This will open the trips popup, which will contain information on the imported trip. Features of the trip details window are\+:


\begin{DoxyItemize}
\item An events trend, heights trend, temperature trend, and throttle trend, which can be toggled between with the middle button cluster.
\item Trip details and statistics, such as the number of gathered points during the flight or the sample rate are located in the left most text box.
\end{DoxyItemize}



{\itshape The plot can be changed to display different information by selecting the buttons from the main cluster}\hypertarget{md__h__repos__argous_src_esp32_as7_gamma__off__board__software_autotoc_md15}{}\doxysubsection{Using the 3\+D Scatter Plot}\label{md__h__repos__argous_src_esp32_as7_gamma__off__board__software_autotoc_md15}
The 3D Scatter plot shows a representation of the points collected by the drone based on sensor position.

It can be accessed by going to “\+Trip Details” and then pressing on “\+Display 3D Point Cloud Model”



{\itshape Pressing ‘\+Display 3D Point Cloud Model’ will open the visualisation of the data points}

\hypertarget{md__h__repos__argous_src_esp32_as7_gamma__off__board__software_autotoc_md16}{}\doxysection{Exporting 3\+D Map}\label{md__h__repos__argous_src_esp32_as7_gamma__off__board__software_autotoc_md16}
The data can be exported by pressing “\+Export” on the home page or by going to ‘\+File \texorpdfstring{$>$}{>} Export PLY’ in the toolbar.

The exported file is the most recent 3D Model. If the new trip data has not been imported yet, the exported model will be loaded from the previous trip.

\hypertarget{md__h__repos__argous_src_esp32_as7_gamma__off__board__software_autotoc_md17}{}\doxysection{Software Features and Application Data}\label{md__h__repos__argous_src_esp32_as7_gamma__off__board__software_autotoc_md17}
The Base Station Software is designed as a support tool for the main code modules with \mbox{\hyperlink{namespace_a_s7}{AS7}}. Information related to the running application can be found in the {\ttfamily appdata} repository, mostly containing temporary data. This includes\+:


\begin{DoxyItemize}
\item The default drone configuration
\item The current drone details
\item A copy of the most recent 3D Point Cloud
\item A text file containing all recent file locations
\end{DoxyItemize}

The files can be edited and are all encoded as text files.\hypertarget{md__h__repos__argous_src_esp32_as7_gamma__off__board__software_autotoc_md18}{}\doxysubsection{Lifetime measurements}\label{md__h__repos__argous_src_esp32_as7_gamma__off__board__software_autotoc_md18}
Lifetime measurements aim to provide the flight history for the drone. On every import, the statistics are updated. They can be found under “\+Running Drone Stats” on the homepage.
